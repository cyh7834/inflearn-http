## 섹션 3. HTTP 기본
### 1. HTTP 특징
* **모든 것이 HTTP**: HTML, 이미지, 영상, JSON 등 모든 데이터를 전송.
* **클라이언트 서버 구조**: 클라이언트는 요청을 보내고 기다리며, 서버는 응답을 처리하는 구조로 분리되어 각각 독립적 진화 가능.
* **Stateless (무상태성)**:
    * 서버가 클라이언트의 이전 상태를 보존하지 않음.
    * 장점: 아무 서버나 호출해도 되므로 서버 증설(Scale-out)이 매우 유리함.
    * 단점: 로그인 유지 같은 상태를 위해 쿠키나 세션 등을 추가로 사용해야 함.
* **비연결성**: 요청을 주고받은 후 연결을 바로 끊어 서버 자원 소모를 최소화함. (현재는 성능을 위해 지속 연결(Persistent Connections) 사용)
