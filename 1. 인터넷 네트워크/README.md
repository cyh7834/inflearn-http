## 섹션 1. 인터넷 네트워크
### 1. IP (인터넷 프로토콜)
* **역할**: 복잡한 인터넷 망에서 메시지를 목적지까지 전달하기 위한 규칙.
* **IP 패킷**: 출발지 IP, 목적지 IP, 전송 데이터 등을 포함한 데이터 단위.
* **한계**:
    * **비연결성**: 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷을 전송함.
    * **비신뢰성**: 중간에 패킷이 사라지거나(Packet Loss), 패킷이 순서대로 도착하지 않을 수 있음.
    * **프로그램 구분 불가**: 같은 IP를 사용하는 서버 내에서 여러 애플리케이션이 동작할 때 구분이 어려움.

### 2. TCP / UDP
* **TCP (Transmission Control Protocol)**: 전송 제어 프로토콜.
    * **3-way handshake**: 접속 전 서버와 클라이언트가 연결을 확인하는 과정 (SYN -> SYN+ACK -> ACK).
    * **데이터 전달 보증**: 데이터 전송 시 서버로부터 잘 받았다는 응답을 받음.
    * **순서 보장**: 패킷 순서가 잘못되면 재전송을 요청함.
    * 신뢰성이 중요할 때 사용 (대부분의 HTTP 통신).
* **UDP (User Datagram Protocol)**: 사용자 데이터그램 프로토콜.
    * TCP의 3-way handshake나 순서 보장 기능이 없어 매우 단순하고 빠름.
    * 최근 HTTP/3에서 성능 최적화를 위해 도입됨.
